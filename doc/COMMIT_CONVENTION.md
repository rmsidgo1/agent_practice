# Commit Convention

## 커밋 메시지 구조

```
<type> subject

[body]

[footer]
```

---

## Type 종류

| Type       | 설명                                  |
|------------|---------------------------------------|
| `feat`     | 새로운 기능 추가                       |
| `fix`      | 버그 수정                             |
| `docs`     | 문서 수정 (README 등)                 |
| `style`    | 코드 포맷 변경 (기능 변화 없음)        |
| `refactor` | 코드 리팩토링                         |
| `test`     | 테스트 코드 추가 또는 수정            |
| `chore`    | 빌드 설정, 패키지 관리 등 기타 변경   |

---

## 규칙

1. `subject`는 50자 이내로 작성한다.
2. `subject`는 명령형으로 작성하고 마침표를 붙이지 않는다.
3. `body`는 선택 사항이며, 변경 이유와 내용을 설명한다.
4. `footer`는 선택 사항이며, 관련 이슈 번호를 참조한다. (예: `Closes #123`)
5. 커밋 메시지는 **한글**로 작성한다.
6. 헤더의 type은 `< >` 문자로 감싸서 표기한다. (예: `<fix> 긴급 Fix`)

---

## 예시

```
<feat> 로그인 기능 추가

사용자 이름과 비밀번호로 로그인할 수 있는 기능을 추가했습니다.

Closes #12
```

```
<fix> 긴급 Fix
```

```
<fix> 회원가입 시 이메일 중복 체크 오류 수정
```

```
<docs> README 설치 방법 업데이트
```
